# 排球小游戏项目说明书 (Java 版)

## 1. 项目概述
开发一款 **3vs3 排球小游戏**，采用 Java 实现，包含图形化交互界面，并提供一个用 HTML + JavaScript + Canvas 制作的网页版。玩家以俯视 45 度固定视角观看整个球场，可操控己方随机一名球员，与 AI 控制的其他 5 名球员进行对抗。先得 5 分的一方获胜。

### v2.2 新增功能
- **玩家登录界面**：启动后输入选手名，名字用于加油气泡文字
- **三种难度模式**：简单/普通/困难，影响双方 AI 犯错概率；简单模式我方更稳、敌方更容易失误，普通模式均衡，困难模式则相反，但整体 AI 不会太弱
- **BGM 背景音乐**：排球少年 OP2《FLY HIGH!!》循环播放（JavaFX MediaPlayer），从模式选择界面开始
- **裁判系统**：黑衣黑帽裁判坐在绿色高椅上（X≈-0.6, Y=9），得分时平举手臂指向得分方半场 + 吹哨动作（哨子从胸前移到嘴边）+ 播放哨声音效，挥手动画持续1.5秒
- **观众席+加油气泡**：场地四侧阶梯看台 + ~200名彩色衣服观众坐在绿色椅子上。得分时在观众席区域弹出白色圆角气泡框。我方得分显示夸赞鼓励语（如"xx太厉害了"、"xx勇敢飞，我们永相随"等9种），敌方得分显示安慰勉励语（如"xx别放弃啊"、"xx你一定可以，相信自己"等9种）
- **黄蓝排球纹理**：模拟真实 Mikasa 排球，5块黄蓝交替弧形面板(clip+fillOval)+4条二次贝塞尔接缝线+高光
- **球空间感增强**：阴影近大远小，但无论多高都有最小影子点（最小宽度6px，最小透明度0.10），离地高度数值显示（如"2.3m"）、接触地面加深阴影
- **AI拦截点预测**：跑向 z=1.0m 击球高度拦截点（非落地处），命中率大幅提升
- **等距投影逆变换**：鼠标→世界坐标精确映射（`A=(sx-OX)/(COS30×SCALE), X=(A+B)/2`），身体旋转准确跟随鼠标
- **发球随机化**：每次发球水平速度 3.5±1.0 m/s，方向偏转 ±15°，竖直速度 5.0~6.0 m/s
- **AI犯错冷却机制**：单次犯错冷却 0.5 秒（`aiBlunderCooldown`），避免每帧重试导致犯错率失效
- **AI不抢玩家球**：我方AI在球落点距离玩家2.5m以内时不接球，确保玩家有充足的触球机会
- **网页版同步版本**：`volleyballGame-online/index.html` 使用原生 Canvas + JavaScript 复刻主要逻辑，包含三种难度、AI、裁判、观众席和加油气泡
- **360°全方位头部**：正面完整五官（双眼+瞳孔高光+眉毛+鼻子+嘴巴），背面后脑勺头发+高光，侧面单眼+耳朵+鼻子轮廓
- **身体旋转系统**：所有部位通过 `pWorld()` 旋转矩阵按 `facingX/facingY` 绕中心轴旋转，躯干宽度随视角动态变化
- **哨声音效**：得分时播放裁判哨声（referee's whistle.mp3）

## 2. 技术栈
- **语言**：Java 17+
- **图形框架**：**JavaFX 21** (Canvas 2D 等距45°投影渲染，兼容集成显卡)
- **构建工具**：Maven（推荐 3.6+）
- **开发环境**：VS Code + Cline 插件（调用 AI 辅助开发）
- **核心依赖**：JavaFX 库 21.0.2（`javafx.controls`, `javafx.graphics`, `javafx.media` 等）

> **说明**：本项目采用 JavaFX Canvas 2D 等距投影实现仿3D效果，不依赖 OpenGL/D3D 3D 管线，兼容包括 AMD Radeon 610M 在内的各种集成显卡。所有场景元素通过 GraphicsContext 逐帧绘制。网页版则使用原生 HTML5 Canvas 与 JavaScript 复刻同样的规则与表现。

## 3. 世界坐标系与场景定义

### 3.1 坐标系
采用 **右手系**（JavaFX 默认）：
- **X 轴**：场地宽度方向，范围 0 ～ 9 m
- **Y 轴**：场地长度方向，范围 0 ～ 18 m（0 为我方底线，9 为球网，18 为对方底线）
- **Z 轴**：垂直高度（向上为正），地面为 Z=0

> 实际 JavaFX 中通常 Y 轴指向上方，Z 轴指向屏幕外。所有位置、速度皆指逻辑坐标，具体的等距投影转换在 `GameRenderer.p()` 中处理。

### 3.2 场地尺寸
- 排球场：长 18m（Y方向），宽 9m（X方向）
- 地板颜色：黄色（模拟木地板），边界使用白色线条
- 球网：位于 Y = 9m 处，沿 X 方向延伸，高度 2.43m，网厚视为 0
- 我方半场：Y ∈ [0, 9] ，对方半场：Y ∈ (9, 18]
- 边界判断：球落在 X ∈ [0, 9] 且 Y ∈ [0, 18] 以内为界内，否则出界
- 场地四周阶梯式观众看台（上下各4级阶梯，左右各4级阶梯）

### 3.3 球员参数
- 球员身高：1.9 m（头顶高度）
- 肩膀高度（手臂转动点）：1.8 m
- 臂长：0.6 m
- 球员初始位置（三角形站位）：
  - 我方：`(4.5, 7)`（前排一人），`(2, 4)`（后排左），`(7, 4)`（后排右）
  - 敌方：`(4.5, 11)`（前排），`(2, 14)`（后排左），`(7, 14)`（后排右）
- 球员默认朝向：我方朝向 +Y 方向（面对对方），敌方朝向 -Y 方向

### 3.4 球的参数
- 排球的半径约 0.1 m
- 黄蓝交替弧形面板纹理（5块面板clip+4条缝线），模拟真实 Mikasa 排球
- 球阴影近大远小（最小宽度6px，最小透明度0.10，永不消失）、高度数值显示、接触地面加深阴影

## 4. 画面渲染与视角

### 4.1 视角固定
采用 Canvas 2D 等距45°投影俯瞰整个 18m×9m 场地，画面固定不随人物移动：
- 投影公式：`sx = OX + (X + Y) × cos30° × SCALE`, `sy = OY + (X - Y) × sin30° × SCALE - Z × SCALE`
- 缩放比例 SCALE = 36 px/m，使全场完整显示在 1280×720 窗口中
- 始终保持固定投影，不随游戏移动。

### 4.2 场景元素绘制
- **地面**：用平行四边形填充表示半场区域，我方浅黄、对方稍深黄，外围深绿色。
- **边线**：白色线条描边（外框+中线+进攻线）。
- **球网**：半透明矩形面 + 5条横线 + 8条纵线 + 两根立柱 + 柱顶球 + 红色天线。球碰网时有0.5m形变动画。
- **球员**：
  - 身体所有部位通过 `pWorld()` 函数围绕身体中心轴（facingX/facingY）旋转：`worldX = x + lx*facingY + ly*facingX`
  - 360°全方位头部——正面双眼/眉毛/鼻子/嘴巴，背面后脑勺头发+高光，侧面单眼+耳朵，根据 bodyAngle 的 cos/sin 分量平滑过渡
  - 躯干宽度随视角动态变化（正面最宽、侧面最窄）
  - 球衣号码仅在正面面对镜头时显示
  - 球衣颜色正面亮色/背面暗色
  - 手臂分上臂短袖+前臂，击球时有顺时针/逆时针挥臂动画+手掌
- **球**：黄蓝交替弧形面板 + 4条二次贝塞尔接缝线 + 白色高光 + 暗色边框。
- **裁判**：黑衣黑帽，坐在绿色高椅上（X≈-0.6, Y=9），胸前挂银色哨子。得分时一手平举指向得分方半场，另一手拿哨子到嘴边吹哨，挥手动画持续1.5秒。
- **观众席**：四侧阶梯看台（4级阶梯，交替深浅灰色台阶面+深色竖面）+ ~200名彩色衣服观众（7种颜色）坐在绿色椅子上，得分时弹出白色圆角气泡。
- **操控光环**：双层脉冲金色圆环（外层固定+内层呼吸动画）。

### 4.3 视觉立体感
- 使用等距投影自然产生的近大远小透视效果。
- 所有物体带地面椭圆阴影。
- 球阴影近大远小（最小6px，alpha最小0.10，再高也永不消失），离地高度数值显示（如"2.3m"），接触地面时加深阴影。
- 敌方球员在网后渲染时加半透明遮挡效果（alpha=0.72）。
- 球在网后时同样加半透明处理（低于网高时 alpha=0.7）。

### 4.4 渲染顺序（从后到前）
```
1. 观众席+观众（最底层，场外）
2. 裁判（含挥手动画）（场外中线左侧）
3. 加油气泡（场外观众席上方）
4. 敌方球员（半透明 alpha=0.72）
5. 球网后层（立柱底面）
6. 球在对方半场（低于网高则 alpha=0.7）
7. 球网前层（网面+网线+立柱+天线）
8. 我方球员（全透明，不被遮挡）
9. 球在我方半场（全透明，画在最上层）
10. 操控光环（最顶层）
```

## 5. 游戏规则与得分

### 5.1 得分条件
1. 球落在**对方场地界内**：我方得 1 分。
2. 球落在**我方场地界内**：对方得 1 分。
3. 任意一方将球打出**全场边界**（X∉[0,9] 或 Y∉[0,18]）：最后触球方的对方得分。
4. 若球在飞行中碰到球网后落入本应出界的区域，判罚同上。

### 5.2 胜利条件
先得 **5 分** 的球队获胜，游戏结束，显示比分并提示按空格重新开始（重新开始后比分重置为0-0）。

### 5.3 发球与球权
- 得分方获得下一次发球权。
- 开局第一次发球，随机决定球权，由发球方后排右方球员发球。
- 发球具有随机性：水平速度 3.5 ± 1.0 m/s，方向偏转 ±15°，竖直速度 5.0~6.0 m/s。

## 6. 玩家控制

### 6.1 操控球员选取
每局（每分）开始或重新发球时，系统从**我方三名球员**中随机选择一名交给玩家控制，其余五名由 AI 控制。选定后该球员上方显示金色光环指示器，直到下一次得分后重新随机选取。

### 6.2 键盘操作
- **移动**：绝对方向（平行于球场轴线，不依赖人物朝向）  
  - `W` 或 `w`：向敌方前进（+Y方向）  
  - `S` 或 `s`：向我方后退（-Y方向）  
  - `A` 或 `a`：向左移动（-X方向）  
  - `D` 或 `d`：向右移动（+X方向）  
  移动速度：**5 m/s**（每帧更新位置）。
- **跳跃**：`空格` 键，最高跳跃约 **2.6 m**（g_player=8 m/s²，JUMP_VZ≈6.45 m/s）。
- **调整击球水平速度**：数字键 `0` ～ `9`，按下后设定玩家下次击球的**水平速度大小 V**（m/s）。默认值：V=4。
- **ESC**：返回模式选择界面。

### 6.3 鼠标操作
- **角色朝向**：使用等距投影逆变换将屏幕坐标 (sx, sy) 精确转换为世界坐标 (X, Y)：
  - A = (sx - OX) / (COS30 × SCALE) = X + Y
  - B = (sy - OY) / (SIN30 × SCALE) = X - Y
  - X = (A + B) / 2, Y = (A - B) / 2
  球员朝向向量 = normalise(mouseX - playerX, mouseY - playerY)。
  身体以 12 rad/s 角速度平滑旋转至目标朝向。
- **传球**：`鼠标左键` 单击。若当前球处于可击球范围内，则执行传球动作。
- **扣球**：`鼠标右键` 单击。条件同上，执行扣球动作。

### 6.4 击球条件（玩家与 AI 共用）
当球满足以下**全部**条件时，球员可击球：
1. 球的空间位置到球员**身体中心位置**（逻辑坐标 (x, y, z+1.0)）的三维距离 **≤ 0.9 m**（AI）/ **≤ 1.05 m**（玩家传球）/ **≤ 1.35 m**（玩家扣球），确保近网高球更易被玩家扣杀。
2. 球的方位相对于球员正面朝向的角度在 **[30°, 150°]** 之间（球员前方 120° 范围）。

> 注意：用离地1m的身体中心作为检测原点，半径扩大到0.9m，确保前方地面落球也可接起。球员跳跃时 Z 坐标变化，中心位置随之变化。

## 7. 物理引擎

### 7.1 球的运动
球的状态由位置 (x, y, z) 和速度 (vx, vy, vz) 表示。每帧更新时间步长 Δt（约 0.016s，对应 60fps）：
- 水平速度恒定：`vx`, `vy` 不变（忽略空气阻力）
- 垂直速度受重力影响：`vz_new = vz_old - g * Δt`，其中 **球的 g = 4 m/s²**
- 位置更新：x_new = x_old + vx * Δt, y_new = y_old + vy * Δt, z_new = z_old + vz_old * Δt

### 7.2 击球速度设置
击球瞬间，球的新速度根据击球类型设定：
- **传球**：水平速度矢量 = 玩家朝向 × V_set（默认3 m/s），垂直速度 `vz = +5 m/s`
- **扣球**：水平速度矢量 = 玩家朝向 × V_set（默认4 m/s），垂直速度 `vz = -0.4 m/s`（较缓的下落弧线，配合水平速度×1.5，球可飞得更远）

玩家通过 0~9 键改变 V_set 值。AI 击球扣球 5 m/s、传球 4 m/s。

### 7.3 球员跳跃物理
`vz_player = sqrt(2 × 8.0 × 2.6) ≈ 6.45 m/s`。球员跳跃使用独立的 **g_player = 8 m/s²**。

### 7.4 球网碰撞
当球穿越 Y=9 且 `z < 2.43m` 时：球弹回，vy 反向 ×0.2，vz ×0.7，产生 0.5m 网形变动画。

### 7.5 球的阴影
球阴影位于球的正下方地面，采用近大远小效果显著变化：阴影半径 `shadowR = max(3, r×(1.5 - h×0.45))`，即高h=0时影子半径12px（直径24px≈球直径的1.5倍），随高度升高影子线性缩小（最小3px）。透明度 `shadowAlpha = max(0.06, 0.55/(1+h×0.35))` 随高度逐渐变淡。球接近地面（h<0.8m）时额外叠加加深阴影圈，半径增大40%，透明度从0线性增至0.30，增强落地感且落地时球底部与影子重合。

### 7.6 出界与落地检测
- **界内落地**：当球的 `z <= 0.1`，且 `0 <= x <= 9` 且 `0 <= y <= 18`。
- **出界**：球触地时超出范围，得分判给最后非触球方。

## 8. AI 控制设计（v2.2 增强版）

### 8.1 决策优先级
```
优先级 1：紧急接球（被分配为接球者时）→ 跑向 z=1.0m 拦截点
优先级 2：防守跑位（球飞行中但非接球者）→ 向球方向靠拢40%
优先级 3：回位（球不在飞行中）→ 返回默认三角站位
```

### 8.2 拦截点预测
使用 `predictAtHeight(1.0m)` 而非 `predictLand()`，计算球降到击球高度时的世界坐标，AI 跑向拦截点而非落地处（落地时球低于可击球范围）。

### 8.3 击球决策
- 距网 < 1.8m 且球高 > 1.8m → 扣球（水平速度 4.0~7.0 m/s 随机）
- 其他情况 → 传球（水平速度 3.0~5.5 m/s 随机）
- 在对方半场使用 9×5 网格搜索最远空档点作为击球目标

### 8.4 AI 犯错机制（冷却式）
AI 首次进入击球范围时按难度概率决定是否犯错，犯错后冷却 0.5 秒（`aiBlunderCooldown`），期间不再尝试击球。避免每帧重试导致犯错率形同虚设。

| 难度 | 我方 AI | 敌方 AI |
|------|--------|--------|
| 简单 | 12% | **28%** |
| 普通 | 18% | 18% |
| 困难 | 28% | **12%** |

> 网页版同样按这三档概率执行，保持“简单我方更稳、困难敌方更强”的方向，但不会把 AI 削得太弱。

### 8.5 跳跃拦截
球高 > 2.0m 且球员能够到球+0.5m 高度时跳跃拦截。球低于 0.2m 时不跳跃（不无谓追球）。

### 8.6 玩家优先接球
我方 AI 在球落点距离人类玩家 **2.5m 以内**时，不分配接球任务，确保玩家角色有充足的触球机会。

### 8.7 网页版说明
- 网页版入口为 `volleyballGame-online/index.html`。
- 网页版与 Java 版保持相同的场地尺寸、球网高度、AI 难度和比分规则。
- 网页版的 AI 已按 Java 原版修正接球者分配、击球判定和犯错机制。

## 9. 游戏流程与状态管理

### 9.1 启动流程
```
登录界面（输入选手名 + 验证）
    ↓ 点击"进入游戏"
模式选择界面（简单/普通/困难）+ BGM开始播放
    ↓ 选择模式
正式游戏界面（按空格开始，ESC返回模式选择）
```

### 9.2 状态机
- **WAITING**：等待玩家按空格开始比赛
- **PLAYING**：回合进行中
- **SCORE**：刚刚得分，暂停2秒（显示裁判挥手+加油气泡+哨声）
- **GAME_OVER**：一方达5分，按空格重新开始（比分重置为0-0）

### 9.3 回合流程
1. 得分后重置所有球员到初始站位，随机选择玩家操控角色
2. 由得分方后排右球员发球（速度/方向随机化）
3. 玩家和 AI 按照规则对抗，直到球落地/出界
4. 判断得分，更新比分，裁判挥手+吹哨，加油气泡弹出（我方得分→夸赞语，敌方得分→安慰语），暂停2秒
5. 检查是否 5 分达成

## 10. 界面与交互细节

### 10.1 UI 元素
- **登录界面**：选手名输入框 + 非空/最长10字验证
- **模式选择界面**：模式名称 + 说明 + 选手名 + 返回按钮
- **游戏界面**：窗口正上方居中显示比分/速度/状态/控制球员/选手名（分行排列，不遮挡场地）
- 加油气泡（2秒，出现在观众席区域，我方得分=夸赞，敌方得分=安慰）+ 裁判挥手（1.5秒）+ 裁判哨声

### 10.2 加油气泡文字系统
得分时气泡文字根据得分方不同：

| 得分方 | 文字数组 | 示例 |
|--------|---------|------|
| 我方得分(0) | `CHEER_WIN_TEXTS` (9种) | "xx太厉害了！"、"xx勇敢飞，我们永相随！"、"xx你是最棒的！"、"xx打败对面啊！"、"xx不愧是排球之神！"等 |
| 敌方得分(1) | `CHEER_LOSE_TEXTS` (9种) | "xx再加把劲！"、"xx你一定可以，相信自己！"、"xx别放弃啊！"、"xx别打假球啊！"、"xx不要气馁！"等 |

### 10.3 音乐与音效
- BGM：排球少年 OP2《FLY HIGH!!》（JavaFX MediaPlayer），从模式选择界面循环播放，音量 35%
- 哨声：得分时播放 referee's whistle.mp3（JavaFX MediaPlayer），音量 60%
- ESC 退出游戏时自动停止BGM

### 10.4 相机固定
相机不移动，确保全程可见整个球场。

## 12. 等距投影公式

```
屏幕坐标:
  sx = OX + (X + Y) × cos30° × SCALE
  sy = OY + (X - Y) × sin30° × SCALE - Z × SCALE

参数: SCALE = 36 px/m, COS30≈0.866, SIN30=0.5, OX=200, OY=530

逆变换（屏幕→世界）:
  A = (sx - OX) / (COS30 × SCALE) = X + Y
  B = (sy - OY) / (SIN30 × SCALE) = X - Y
  X = (A + B) / 2,  Y = (A - B) / 2
```
