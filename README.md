# 🏐 3v3 排球小游戏

基于 **Java 17 + JavaFX 21 + 2D Canvas** 开发的排球对抗小游戏。采用等距45°俯视角渲染，兼容集成显卡。玩家操控己方一名球员，与 5 名 AI 球员展开 3v3 排球对抗，先得 5 分者获胜。

## 📖 项目概述

本游戏模拟了一场 3 对 3 的室内排球比赛。游戏场景为 **18m × 9m** 标准排球场，场地中央设置 **2.43m** 高球网将其分为两个半场。所有 6 名球员身高均为 **1.9m**，臂长 **0.6m**，具备跳跃和击球能力。

> **实现方案**：采用 JavaFX Canvas 2D 等距投影实现仿3D效果，不依赖 OpenGL/D3D 3D 管线，兼容包括集成显卡在内的任何设备。球员具备完整建模（头发、五官、球衣号码等），球网有碰触形变效果。

## ✨ 功能特性

- **玩家登录界面**：启动后输入选手名，名字用于比赛中加油气泡文字
- **三种难度模式**：简单/普通/困难，影响双方 AI 犯错概率
- **等距45°俯瞰**：平行四边形仿3D，固定画面囊括全场，不依赖3D管线
- **完整球员建模**：360°全方位头部——正面双眼/眉毛/鼻子/嘴巴，背面后脑勺头发，侧面单眼+耳朵，随身体朝向平滑过渡
- **丝滑转身**：等距投影逆变换实现鼠标→世界坐标精确映射，身体平滑旋转至鼠标指向方向（12 rad/s角速度）
- **WASD 绝对移动**：平行球场轴线，移速 5 m/s，空格跳跃约 0.8m
- **球网形变效果**：碰网后 0.5m 凹陷动画 + 反弹衰减 + 网后敌方半透明遮挡
- **独立重力系统**：球 g=4 m/s²，球员跳跃 g=5 m/s²
- **黄蓝排球纹理**：模拟真实 Mikasa 排球，5 块黄蓝交替弧形面板 + 4 条接缝线 + 高光
- **球空间感增强**：阴影近大远小（再高都有最小影子点，永不消失）、离地高度数值显示（如"2.3m"）、接触地面加深阴影
- **AI 智能对手**：拦截点预测（z=1.0m 最佳击球高度）+ 9×5 网格搜索空档 + 跳跃拦截
- **裁判系统**：黑衣黑帽裁判坐在绿色高椅上（中线左侧），得分时平举手臂指向得分方半场 + 吹哨动作（哨子从胸前移到嘴边）+ 播放哨声音效
- **观众席+加油气泡**：四侧阶梯看台 + ~200 名彩色观众坐在绿色椅子上。得分时在观众席区域弹出白色圆角气泡框。我方得分时说鼓励夸赞的话（如"xx太厉害了"），敌方得分时说安慰鼓励的话（如"xx别放弃啊"）
- **BGM 背景音乐**：排球少年 OP2《FLY HIGH!!》循环播放（从模式选择界面开始）
- **哨声音效**：得分时播放裁判哨声
- **随机角色分配**：每回合随机指定玩家操控我方的 1 名球员
- **发球随机化**：每次发球速度 ±1 m/s，方向 ±15°，竖直速度也微调
- **完备得分系统**：界内落地得分 + 出界判罚 + 5 分制胜利
- **实时 UI**：窗口正上方居中显示比分、击球速度 V、控制球员提示、游戏状态、选手名（分行排列）
- **AI不抢玩家球**：我方AI在球落点距离玩家2.5m以内时不抢球，确保玩家的触球机会

## 📐 玩法规则

### 开局站位（三角形阵型）

| 球员 | 阵营 | 坐标 (X, Y) | 位置描述 |
|------|------|------------|----------|
| 我方前排 | 🔵 | (4.5, 7) | 靠近球网 |
| 我方后排左 | 🔵 | (2, 4) | 左下角 |
| 我方后排右 | 🔵 | (7, 4) | 右下角 |
| 敌方前排 | 🔴 | (4.5, 11) | 靠近球网 |
| 敌方后排左 | 🔴 | (2, 14) | 左上角 |
| 敌方后排右 | 🔴 | (7, 14) | 右上角 |

### 得分条件

| 情况 | 结果 |
|------|------|
| 球落在**对方半场**界内 (Y ∈ [9, 18], X ∈ [0, 9]) | ✅ 我方得 1 分 |
| 球落在**我方半场**界内 (Y ∈ [0, 9], X ∈ [0, 9]) | ❌ 敌方得 1 分 |
| 球出界 (X ∉ [0, 9] 或 Y ∉ [0, 18]) | 最后触球方对方得分 |
| 球触网后落在界外 | 按最后触球方判定 |

### 胜利条件

**先得 5 分** 的球队获胜，显示胜利提示，按空格键重新开始（比分重置为0-0）。

### 发球规则

- 得分方获得下一次发球权
- 开局首次发球权随机决定
- 由后排右球员发球
- 发球具有随机性：水平速度 3.5±1.0 m/s，方向偏转 ±15°，竖直速度 5.0~6.0 m/s

### 角色分配

- 每回合（每次得分后）系统**随机**从我方 3 名球员中选 1 人交由玩家控制
- 其余 5 名球员全部由 AI 自动操作
- 被选中的球员头顶显示金色光环

## ⚙️ 物理系统

### 球的运动

采用简化的抛物运动模型：

```
水平速度恒定：vx = const, vy = const (忽略空气阻力)
竖直速度：vz(t) = vz₀ - g·t
位置更新：
  x(t) = x₀ + vx·t
  y(t) = y₀ + vy·t
  z(t) = z₀ + vz·t
```

> 注：球的重力 g = 4 m/s²，球员跳跃使用独立重力 g_player = 5 m/s²。

| 参数 | 值 | 说明 |
|------|-----|------|
| 球重力加速度 g | **4 m/s²** | 竖直向下 |
| 球员跳跃重力 | **8 m/s²** | 跳跃专用，独立系统 |
| 球员移速 | 5 m/s | WASD 水平移动 |
| 球半径 | 0.1 m | 近似标准排球尺寸 |
| 帧率 | ~60 FPS | Δt ≈ 0.016s |
| 球网高度 | 2.43 m | 位于 Y = 9 |

### 击球条件

球员可以击球需**同时满足**以下两个条件：

1. **距离条件**：球到球员身体中心 (z + 1.0m) 的三维距离 ≤ **0.9m**（AI）/ **1.05m**（玩家传球）/ **1.35m**（玩家扣球），含手臂伸展+弯腰范围
2. **角度条件**：球相对于球员正面的方位角 ∈ **[30°, 150°]**（前方 120° 扇形区域）

### 击球类型

| 击球类型 | 触发方式 | 水平速度 V | 竖直速度 Vz | 击球方向 |
|----------|----------|-----------|------------|----------|
| **传球** | 鼠标左键 | 默认 4 m/s（可用 0~9 调整） | **+5 m/s** 斜向上 | 球员正面方向 |
| **扣球** | 鼠标右键 | 默认 4 m/s（可用 0~9 调整） | **-0.4 m/s** 斜向下 | 球员正面方向 |

> 玩家可通过数字键 `0` ~ `9` 实时调整 V 值，传球与扣球共用该设定。

### 跳跃物理

```
跳跃最高高度：约 1.2m
跳跃初速度：vz = √(2 × 8.0 × 1.2) ≈ 4.38 m/s
空中运动：vz -= 8.0·dt, z += vz·dt
落地条件：z ≤ 0 → z = 0, vz = 0
```

### 球网碰撞

当球穿越 Y = 9 平面且球的高度 **z < 2.43m** 时，发生球网碰撞：

- 球弹回（Y 坐标回弹到网线附近）
- 水平速度 `vy` 反向并乘以衰减系数 **0.2**（几乎不反弹）
- 竖直速度乘以衰减系数 0.7，碰网后主要表现为**竖直下落**
- 产生 **0.5m 网形变**动画效果

### 球的阴影

球阴影位于球的正下方地面，采用近大远小效果显著变化：球高h=0时影子半径12px（直径24px，约为球直径1.5倍），随高度升高影子线性缩小至最小3px，透明度逐渐变淡。球接近地面（h<0.8m）时额外叠加加深阴影圈增强落地感，落地时底部与影子重合。球始终在地面有可见的阴影标记，永不消失。

### 出界检测

球触地时（z ≤ 0.1m）：

| 判定 | X 范围 | Y 范围 | 结果 |
|------|--------|--------|------|
| 界内 | [0, 9] | [0, 18] | 按所在半场判定得分方 |
| 出界 | 超出以上范围 | - | 最后触球方对方得分 |

## 🤖 AI 系统设计

AI 采用**基于规则的分层状态机**，使用**拦截点预测**而非落地预测，命中率显著提升。

### 决策优先级

```
优先级 1：紧急接球（被分配为接球者时）
    ↓
优先级 2：防守跑位（球飞行中但非接球者）
    ↓
优先级 3：回位（球不在飞行中，返回默认站位）
```

### 接球者分配

1. **预测拦截点**：计算球降到 z=1.0m（最佳击球高度）时的世界坐标
2. **计算到达时间**：每个 AI 球员到拦截点的距离 ÷ 移动速度
3. **最短时间者负责**：到达时间最短的球员被分配为接球者
4. **玩家优先**：如果球落点距离人类玩家 2.5m 以内，我方 AI 不分配接球任务

### 击球决策

| 条件 | 击球类型 | 水平速度 | 说明 |
|------|----------|----------|------|
| 距网 < 1.8m 且球高 > 1.8m | 扣球 | 4.0~7.0 m/s（随机） | 近网高球扣杀 |
| 其他情况 | 传球 | 3.0~5.5 m/s（随机） | 常规传球过渡 |

### 目标选择

在对方半场使用 **9×5 网格搜索**，找到距离对方所有球员最远的空档点作为击球目标，增加得分概率。

### 跳跃拦截

球高 > 2.0m 且球员能够到球+0.5m 高度时跳跃拦截。球低于 0.2m 时不跳跃。

### AI 犯错机制

采用 **冷却式犯错**：AI 首次进入击球范围时按概率决定是否犯错，犯错后冷却 0.5 秒不再重复尝试。犯错概率受难度模式控制。

## 🎯 难度模式

游戏提供三种难度，影响双方 AI 的犯错概率：

| 难度 | 我方 AI 犯错率 | 敌方 AI 犯错率 | 效果 |
|------|-------------|-------------|------|
| 🟢 **简单** | 8% | **45%** | 敌方接近一半的球漏接，玩家轻松取胜 |
| 🟡 **普通** | 20% | 20% | 双方公平对抗 |
| 🔴 **困难** | 30% | **6%** | 我方漏球较多，敌方精锐 |

### 等距投影公式

```
屏幕坐标:
  sx = OX + (X + Y) × cos30° × SCALE
  sy = OY + (X - Y) × sin30° × SCALE - Z × SCALE

参数: SCALE = 36 px/m, COS30≈0.866, SIN30=0.5, OX=200, OY=530

逆变换（屏幕→世界）:
  A = (sx - OX) / (COS30 × SCALE) = X + Y
  B = (sy - OY) / (SIN30 × SCALE) = X - Y
  X = (A + B) / 2,  Y = (A - B) / 2
```

## ❓ 常见问题

**Q: 运行报 `javafx.*` 找不到类？**
A: 确保使用 `mvn javafx:run` 而非直接 `java` 运行。JavaFX 依赖由 Maven 插件自动加载。

**Q: 为什么不是3D渲染？**
A: AMD Radeon 610M 等集成显卡的 D3D9/D3D11 驱动与 JavaFX 3D 管线不兼容。改为 Canvas 2D 等距投影方案后，所有设备均可100%正常运行。

**Q: 球员移动没有反应？**
A: 确保你操控的球员是头顶有金色光环的那一位。每回合系统随机分配操控角色。

**Q: 球飞得太快/太慢？**
A: 使用数字键 `0`~`9` 调整击球水平速度 V（默认3）。

**Q: 如何让游戏更简单/更难？**
A: 在模式选择界面选择"简单/普通/困难"模式。也可以按 ESC 返回重新选。

**Q: 球在空中看不到位置？**
A: 球旁边会显示实时离地高度（如"2.3m"），且无论多高都有最小影子点在地面标记位置。

**Q: 如何只编译不运行？**
A: `mvn compile` 只编译 Java 源文件。

**Q: BGM 没有声音？**
A: 确保 `src/main/resources/bgm.mp3` 文件存在。如不需要 BGM，可删除该文件或忽略控制台警告。

**Q: GAME_OVER后按空格比分不重置？**
A: 已修复，v2.2后按空格重来时比分正确重置为0-0。

**Q: 我方AI总是抢我的球？**
A: v2.1已优化，球落点在玩家周围2.5m以内时我方AI不接球。
